# Gate C6 sign-off: `renal_adjustment.dfy`

## Purpose

Gate C6 (NL-dialogue confirmation) exists to catch a spec/requirement-text
mismatch at authoring time rather than at review time. Per the
infrastructure plan (`/root/.claude/plans/stateless-weaving-firefly.md`),
this gate is applied earlier in this POC's build order than it was for
`dosage.dfy` — immediately after the spec first verifies clean, not
deferred behind STPs/mutation testing.

## A real generalization gap found and fixed before this summary could be generated

The infrastructure plan assumed `evidence/dafny_nl_summary.py::summarize_method`
"generalizes for free" since it's parameterized by `method_name`. That
was checked, not trusted — and it was wrong in one respect:
`_find_method_header` (the shared helper this module, `dafny_spec_lint.py`,
and `dafny_mutate.py` all depend on) only matched Dafny's `method`
keyword, never `function`. Every real Dafny declaration this repo had
summarized before today was a `method` (`dosage.dfy`'s `ExpectedDose`
function was never passed to this extractor directly) — `renal_adjustment.dfy`
is pure functions, five of them, zero methods, and the gap only surfaced
because this is the first time the tooling was actually pointed at one.

Fixed by widening the shared regex to match `method` or `function`
(`evidence/dafny_spec_lint.py::_find_method_header`, and the duplicate
lookup in `evidence/dafny_mutate.py::_method_header_span` for
consistency) — a small, bounded extension, not a redesign, confirmed by
running the real file through it rather than assumed from the code
reading generic. Two new regression tests
(`tests/test_dafny_nl_summary.py::test_summarizes_a_function_not_just_a_method`,
`tests/test_dafny_spec_lint.py::test_requires_clause_extraction_matches_a_function_not_just_a_method`).

**A second, real bug found in the same pass, not a hypothetical:**
`_REQ_ID_RE`'s character class (`[A-Z0-9-]`) excluded lowercase letters,
so it silently truncated `REQ-RENAL-1a` to `REQ-RENAL-1` — citing the
wrong requirement ID rather than dropping the citation visibly.
`dosage.dfy`'s REQ-IDs (`REQ-GIP-1-4-12`, `REQ-GIP-1-8-1`) never had a
lowercase suffix, so this never fired until a second spec actually
exercised it. Fixed (`[A-Za-z0-9-]`), regression-tested
(`test_lowercase_suffixed_req_id_is_cited_in_full_not_truncated`). Also
added missing `// REQ-RENAL-*` trailing citations to every `ensures`
clause in `renal_adjustment.dfy` itself — without them, every
postcondition below would have read "no requirement cited," understating
real, existing traceability rather than reflecting an actual gap.

142 tests passing after both fixes (up from 138); `renal_adjustment.dfy`
re-verified clean after the citation comments were added (`5 verified,
0 errors` — comments don't affect verification).
